# OctoAcme — Role Collaboration Checklist

## Purpose
Use this checklist to confirm the right roles are involved at each phase of work without creating a heavyweight approval process.

## When to use
- During kickoff and planning
- Before a milestone or release
- When a risk, dependency, or escalation needs clearer ownership
- During retrospectives to check whether handoffs worked as expected

## Lifecycle collaboration checkpoints

### Initiation
- [ ] Product Manager defines the problem, goals, and success metrics
- [ ] Project Manager confirms stakeholders, timeline assumptions, and communication cadence
- [ ] Stakeholder / Executive Sponsor confirms priority and business alignment
- [ ] Engineering Manager / Engineering Lead and Designer / UX Designer are involved early if technical or UX uncertainty is high
- [ ] Support / Customer Success shares customer pain points or launch constraints when relevant

### Planning
- [ ] Developers, QA / Test Engineers, and Designer / UX Designer review backlog items for clarity, feasibility, and testability
- [ ] Engineering Manager / Engineering Lead validates capacity, sequencing, and major technical dependencies
- [ ] Project Manager records owners, milestones, and cross-team risks
- [ ] Product Manager confirms acceptance criteria and trade-offs
- [ ] QA / Test Engineer outlines the initial test approach and release quality concerns

### Execution & Tracking
- [ ] Developers raise blockers, risks, and implementation trade-offs early
- [ ] QA / Test Engineers validate feature behavior, regression scope, and defect severity
- [ ] Designer / UX Designer reviews in-progress experience details when workflows or UI behavior change
- [ ] Project Manager updates status, risks, and decisions in the shared source of truth
- [ ] Engineering Manager / Engineering Lead helps resolve technical blockers and escalates delivery risk as needed

### Release & Deployment
- [ ] Product Manager confirms the release still meets intended customer value
- [ ] QA / Test Engineer confirms smoke test readiness and outstanding quality risks
- [ ] Project Manager coordinates deployment timing, communications, and go/no-go decisions
- [ ] Support / Customer Success receives release notes, FAQs, and escalation guidance
- [ ] Stakeholder / Executive Sponsor is informed or asked for approval when the release has major business impact

### Retrospective & Continuous Improvement
- [ ] Developers, QA / Test Engineers, Project Manager, and Product Manager review what slowed delivery or created rework
- [ ] Support / Customer Success shares post-release themes and customer friction
- [ ] Engineering Manager / Engineering Lead and Designer / UX Designer help identify durable process or design improvements
- [ ] Action items have owners, due dates, and visible follow-up

## Quick handoff prompts
- Is the owner for this decision explicit?
- Does the next role have the context needed to act without rework?
- Are approval points clear for scope, quality, release, and escalation decisions?
- Has customer-facing communication been prepared for affected audiences?
